# Chronicle Maker

Chronicle Maker 是 Chronicle Engine: RPG Framework 的本地数据包编辑器。

它面向不会写 JSON 的整合包作者：打开网页、选择项目、填写表单、点击写入即可生成 CE:RF 所需文件。

## 使用方式

1. 用新版 Edge 或 Chrome 打开 `index.html`。
2. 点击“选择整合包项目文件夹”。
3. 点击“读取已有 CE:RF 配置”可以继续编辑现有内容。
4. 在左侧切换任务、对话、NPC、商店、钱包。
5. 在“导出”页点击“检查数据”。
6. 点击“写入项目”。

任务编辑页顶部有路线图，点击阶段卡片会跳到对应阶段；阶段、目标、跳转、条件、奖励、动作等长配置都可以折叠。

## 推荐输出

默认输出到：

```text
config/chronicle_engine/chronicle_pack
```

这是 CE:RF 的推荐低门槛工作流，不需要理解标准 datapack 目录结构。

也可以切换为标准 datapack 输出。编辑器会在已选择项目下生成：

```text
generated_chronicle_datapack/<namespace>
```

## 自动扫描内容

选择项目后，编辑器会尽量扫描：

- `mods`：根据 jar 文件名推断模组命名空间；
- `kubejs`：从脚本中提取出现过的 `modid:path` ID；
- `config/chronicle_engine/chronicle_pack`：读取已有任务、对话、NPC、商店和钱包。

浏览器无法可靠读取 jar 内部注册表，所以物品和实体下拉建议并不等于完整注册表。需要时仍可手动填入物品 ID。

## 零基础建议

- 任务 ID、对话 ID、商店 ID 推荐使用 `你的命名空间:英文名`。
- 先从“载入示例”开始，确认结构后再改成自己的内容。
- 商店需要先创建钱包货币，否则玩家可能没有可支付的钱币。
- 主线联机同步时，勾选任务里的“团队同步”。

## 支持内容

- 任务：阶段路线图、阶段、击杀、收集、交付、交互、成就目标、阶段奖励、完成奖励。
- 对话：节点、选项、接取任务、打开商店、设置旗标、关闭对话。
- NPC：实体类型绑定、村民职业绑定、名称匹配、NBT 对话字段。
- 商店：分类、商品、花费、获得物品。
- 钱包：一个货币绑定多个物品 ID。
